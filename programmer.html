SAÉ 1.5

Récupérer, traiter et afficher des données automatiquement

Ce qu'on devait faire

Développer un script Python qui va chercher des données CSV sur data.gouv.fr, les traite (calculs de moyennes, valeurs max, filtres) et les envoie automatiquement à un site web pour les afficher sous forme de graphiques.

Ce que j'ai appris

Manipuler des données réelles, c'est bien différent des exercices en cours. J'ai découvert la puissance de Python pour automatiser ce genre de pipeline, et la satisfaction quand les graphiques s'affichent tout seuls après un simple script.

Deuxième année — Scripting avancé et applications réseau
SAÉ 2.1

Scripts d'automatisation et outils offensifs sur l'infrastructure

Ce qu'on devait faire

Sur notre propre infrastructure d'entreprise, on a développé trois types de scripts : automatisation des conteneurs Docker en Bash, interception et reconstitution d'une conversation VoIP avec Python Scapy, et reconnaissance de badges par caméra avec OpenCV.

Ce que j'ai appris

Ce projet m'a montré à quel point le code peut être un outil offensif ET défensif. Reconstituer une conversation VoIP à partir de trames capturées, c'est impressionnant — et ça donne envie de tout chiffrer derrière.

Outils & technologies

Bash & DockerPython ScapyPython OpenCVCapture réseauJournalisation
SAÉ 3.2

Serveur de recherche documentaire en Python

Ce qu'on devait faire

Développer un service client-serveur en Python capable de chercher un mot dans n'importe quel type de document (txt, PDF, HTML, Excel) et d'afficher le contexte autour du résultat.

Ce que j'ai appris

Au-delà de la technique, ce projet m'a appris à coder proprement : respect du PEP 8, docstrings, README en anglais, requirements.txt. Des habitudes qui font la différence en entreprise.

Outils & technologies

Python client-serveurPDF, Excel, HTMLPEP 8 & docstringsGitLabREADME anglais
